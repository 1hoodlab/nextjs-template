# Upgrade dependencies and Tailwind v4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the repo to Node.js 22, upgrade direct dependencies to current major versions (including Next.js 16 and React 19.2.x), migrate Tailwind CSS from v3 to v4 with PostCSS, and pass `npm run build` and `npm run lint`.

**Architecture:** Three phases match the approved spec: (A) pin Node 22 in repo metadata; (B) bump runtime and tooling packages while keeping Tailwind on v3 until the app builds; (C) migrate Tailwind with the official upgrade tool and/or explicit PostCSS + CSS entry changes, then add missing design tokens used by `app/error.tsx` and `app/not-found.tsx` so utilities like `bg-primary` resolve under v4.

**Tech Stack:** Next.js (App Router), React 19, TypeScript, ESLint flat config (`eslint.config.mjs`), Redux Toolkit + redux-saga, PostCSS, Tailwind CSS v4, npm + `package-lock.json`.

### Task 3: Phase C — Tailwind CSS v4 migration

**Files:**
- Modify: `package.json`, `package-lock.json`, `postcss.config.mjs`, `app/globals.css`, `tailwind.config.ts` (may delete)
- Possibly modify: other CSS or config files produced by the upgrade tool

- [ ] **Step 1: Install Tailwind v4 PostCSS packages**

```bash
npm install -D tailwindcss@4.2.2 @tailwindcss/postcss@4.2.2
```

Expected: Exit code 0; `tailwindcss` v4 replaces v3 in `package.json`.

- [ ] **Step 2: Run the official upgrade tool**

```bash
npx @tailwindcss/upgrade@latest
```

The tool migrates configuration toward CSS, updates imports, and adjusts templates. Review the full `git diff` before continuing.

If the tool cannot run (network or environment), apply the manual fallback in **Step 2b** instead of skipping.

- [ ] **Step 2b (fallback only): Manual PostCSS + CSS entry**

Replace `postcss.config.mjs` contents with:

```js
/** @type {import('postcss-load-config').Config} */
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};

export default config;
```

Replace the top of `app/globals.css` — remove:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Add as the first line:

```css
@import "tailwindcss";
```

Keep the existing `:root` and `@media (prefers-color-scheme: dark)` blocks below the import. Remove `tailwind.config.ts` only if v4 no longer needs it after you add `@theme` (see Step 3); otherwise use `@config` in CSS per [Tailwind v4 docs](https://tailwindcss.com/docs/upgrade-guide).

- [ ] **Step 3: Add semantic tokens for `primary`, `primary-foreground`, and `muted-foreground`**

`tailwind.config.ts` previously only extended `background` and `foreground`, but `app/error.tsx` and `app/not-found.tsx` use `text-muted-foreground`, `bg-primary`, `text-primary-foreground`, and `focus-visible:ring-primary`. Define these in `app/globals.css` so Tailwind v4 utilities resolve.

Append after your `:root` block (and mirror values inside the dark `prefers-color-scheme` block for parity):

```css
:root {
  --primary: #171717;
  --primary-foreground: #fafafa;
  --muted-foreground: #737373;
}

@media (prefers-color-scheme: dark) {
  :root {
    --primary: #fafafa;
    --primary-foreground: #0a0a0a;
    --muted-foreground: #a3a3a3;
  }
}
```

Add a Tailwind v4 `@theme` mapping (location: after `@import "tailwindcss"` and alongside existing variable-based colors) so utilities match previous `theme.extend` behavior:

```css
@theme {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --color-muted-foreground: var(--muted-foreground);
}
```

If the upgrade tool already emitted `@theme`, merge these entries into that block instead of duplicating.

- [ ] **Step 4: Update outline utility for Tailwind v4**

In `app/error.tsx` and `app/not-found.tsx`, replace `focus:outline-none` with `focus:outline-hidden` in the long `className` strings on the primary button/link (Tailwind v4 [rename](https://tailwindcss.com/docs/upgrade-guide)).

Example fragment before:

```tsx
focus:outline-none focus-visible:ring-2
```

After:

```tsx
focus:outline-hidden focus-visible:ring-2
```

- [ ] **Step 5: Verify build and lint**

```bash
npm run build
npm run lint
```

Expected: Exit code 0 for both.

- [ ] **Step 6: Manual smoke test**

Run `npm run dev`, open `/`, toggle OS light/dark mode, open a non-existent route for `not-found`, and trigger `error` if feasible. Confirm buttons and text contrast remain acceptable.


Add any other files the upgrade tool modified to the `git add` line before committing.

### Task 4: Optional utility renames (only if visual regression)

**Files:** Search across `app/`, `components/`, `features/`

Tailwind v4 renames some utilities ([shadow / blur / rounded scales, `ring` default width](https://tailwindcss.com/docs/upgrade-guide)). This codebase uses `rounded-*` on buttons but not necessarily the renamed `shadow-sm` → `shadow-xs` pattern everywhere.

- [ ] **Step 1: Search and fix only if something looks wrong after Step 3**

```bash
rg "shadow-sm|outline-none|(?<![-])\\bring\\b" --glob "*.tsx"
```

If `outline-none` remains outside focus rings, evaluate case-by-case: use `outline-hidden` for the old v3 accessibility-preserving behavior, or `outline-none` for true removal per v4.
